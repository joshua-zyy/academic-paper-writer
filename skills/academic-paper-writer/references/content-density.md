# 最低内容密度指南

## substantial draft 判定标准

核心章节已形成可阅读的 prose，而非提纲式句子。以下为最低密度阈值：

| Section | 最低要求 | 过薄信号 |
|---------|---------|---------|
| Introduction | ≥ 4 段，覆盖：问题背景 → 领域现状 → 方法学缺口 → 本文定位与贡献 | ≤ 2 段，或只有背景+贡献列表 |
| Related Work | ≥ 2 个 work clusters，每 cluster 含代表工作+共同局限+与本文关系 | 仅罗列论文名，无分组比较 |
| Method | 按模块分小节，每核心模块含：作用+输入输出+关键公式+设计动机 | 仅概述段，无模块拆解 |
| Experimental Setup | 含数据描述+划分协议+实现细节+风险说明 | 仅参数罗列，无协议与风险 |
| Main Results | 含主表/主结果+协议说明+与 baseline 比较 | 仅数字，无协议与比较上下文 |
| Discussion/Limitations | 含边界分析+失败模式+限制讨论 | 仅"需要未来工作"泛泛而谈 |
| Conclusion | 回扣方法贡献+当前结果边界+下一步关键补强项 | 仅"提出了某方法" |

## 过薄时的处理

若证据不足，优先使用占位符和受限表述把结构写完整：
- `[FIGURE_NEEDED: ...]` — 缺图时在正文占位，不压缩小节
- `[TABLE_NEEDED: ...]` — 缺表时保留表位和分析框架
- `[METHOD_DETAIL_NEEDED: ...]` — 缺方法细节时保留模块结构
- 不因为缺细节就把整节压缩成 2-3 句话

## 量化参考

以下为 approximate 量化参考，用于判定 thin_draft：

| Section | 过薄阈值 | adequate 参考量 |
|---------|---------|----------------|
| Introduction | ≤ 150 词 或 ≤ 2 段 | ≥ 400 词，≥ 4 段 |
| Related Work | ≤ 100 词 或仅 1 个 cluster | ≥ 300 词，≥ 2 clusters |
| Method (核心模块) | ≤ 80 词/模块 | ≥ 200 词/核心模块 |
| Experimental Setup | ≤ 100 词 | ≥ 250 词，含协议与风险 |
| Main Results | ≤ 150 词 或 ≤ 1 表 | ≥ 300 词 + 主表 |
| Discussion/Limitations | ≤ 80 词 | ≥ 200 词 |
| Conclusion | ≤ 60 词 | ≥ 120 词 |

注意：量化参考是辅助信号，不以词数为唯一标准。内容完整性（论证链、协议说明、占位符使用）比词数更重要。

## 扩写原则

- 只用已有证据和合规占位符扩充，不靠臆造补长度
- 优先补足"读者理解链条"：背景 → 问题 → 方法 → 结果 → 边界
- 不堆空泛形容词充数
