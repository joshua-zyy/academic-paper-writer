# 内容密度检查指南（Expansion Pass）

## 目的

Step 10 Expansion Pass 用于检测当前 section 是否属于 "thin draft"，并在可能时补全内容密度缺口。

## Thin Draft 判定标准

| Section | Thin 条件 | 正常标准 |
|---------|----------|---------|
| Introduction | ≤ 2 段，或仅有背景 + 贡献列表 | ≥ 4 段：问题重要性 → 现有缺口 → 本文思路 → 贡献 |
| Related Work | 仅罗列论文名，无 work clusters 或综合比较 | ≥ 2 个 work clusters，每个含比较维度与本文定位 |
| Method | 仅概述段，无模块拆解、公式或设计动机 | 核心模块独立小节，含瓶颈→设计→机制→公式→收益→代价 |
| Experimental Setup | 仅参数列表，无协议、约束或风险说明 | 含数据集描述、划分协议、实现细节、风险标注 |
| Main Results | 仅数字罗列，无与 baseline 的对比分析 | 含结果解读、与基线差距分析、关键观察 |
| Discussion / Conclusion | 仅泛泛结论，无边界分析或失败模式 | 含边界条件、局限性、失败模式、未来方向 |

## 扩写原则

1. **仅使用已有证据和合规占位符**——不为填充篇幅而编造内容
2. **优先补"读者理解链"缺口**——读者从当前文本能否完整理解论证逻辑？
3. **每个扩写段必须承担一个论证职责**——不写空泛铺垫段
4. **Method 扩写优先级**：设计动机 > 核心机制 > 公式 > 标准组件
5. **Results 扩写优先级**：与 baseline 对比分析 > 消融解读 > 效率/定性分析

## 扩写禁忌

- 不得为凑字数而重复已有论述
- 不得在无新证据时将弱 claim 扩写为强 claim
- 不得用华丽 prose 掩盖内容不足
- 不得因篇幅达标就跳过后续 Verification
